This guide will walk you through the essential steps to start using Fitboard: creating your account, logging in, recording your first meal, and viewing your progress data.

## Create Your Account

<Steps>
  <Step title="Register a new account">
    Send a POST request to the registration endpoint with your credentials:

    ```bash
    curl -X POST https://api.fitboard.com/api/v1/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "you@example.com",
        "username": "your_username",
        "password": "your_secure_password"
      }'
    ```

    <Note>
      You can optionally include an avatar URL in your registration request using the `avatar` field, or upload an image file using multipart/form-data.
    </Note>

    The API will return your user profile:

    ```json
    {
      "_id": "507f1f77bcf86cd799439011",
      "email": "you@example.com",
      "username": "your_username",
      "avatar": "https://res.cloudinary.com/ddgf7ijdc/image/upload/v1733781971/userAvatart/Avatars/yf13f4uoy32msk195epl.jpg",
      "isAdmin": false,
      "active": true,
      "createdAt": "2025-11-05T10:30:00.000Z",
      "updatedAt": "2025-11-05T10:30:00.000Z"
    }
    ```
  </Step>

  <Step title="Verify your account is active">
    Your account is created with `active: true` by default, so you can immediately proceed to login.
  </Step>
</Steps>

## Log In to Your Account

Authenticate with your credentials to receive a JWT token:

```bash
curl -X POST https://api.fitboard.com/api/v1/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "your_secure_password"
  }'
```

Successful login returns your token and user profile:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "_id": "507f1f77bcf86cd799439011",
    "email": "you@example.com",
    "username": "your_username",
    "avatar": "https://res.cloudinary.com/ddgf7ijdc/image/upload/v1733781971/userAvatart/Avatars/yf13f4uoy32msk195epl.jpg",
    "isAdmin": false,
    "active": true
  }
}
```

<Warning>
  Store the JWT token securely. You'll need to include it in the `Authorization` header for all subsequent API requests.
</Warning>

## Log Your First Meal

Now that you're authenticated, let's record your first meal. Fitboard supports tracking meals with custom nutrition data.

<Steps>
  <Step title="Prepare your meal data">
    Meals in Fitboard are organized by date and slot (breakfast, lunch, dinner, or snack). Each meal includes detailed nutrition information.
  </Step>

  <Step title="Create a meal entry">
    Use the `/api/v1/meals` endpoint to log your meal:

    ```bash
    curl -X POST https://api.fitboard.com/api/v1/meals \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_JWT_TOKEN" \
      -d '{
        "date": "2025-11-05",
        "slot": "breakfast",
        "customItem": {
          "name": "Oatmeal with Banana",
          "amount": 250,
          "unit": "g",
          "nutritionBasis": {
            "amount": 100,
            "unit": "g"
          },
          "macrosPerBasis": {
            "kcal": 150,
            "protein": 5.5,
            "carbohydrate": 27,
            "fat": 2.5
          }
        }
      }'
    ```

    <Note>
      Fitboard uses a unique constraint on `(userId, date, slot)`, so you can only have one meal per slot per day. Use PATCH to update or the `/move` endpoint to reschedule.
    </Note>

    The response includes your created meal:

    ```json
    {
      "_id": "507f1f77bcf86cd799439012",
      "userId": "507f1f77bcf86cd799439011",
      "date": "2025-11-05",
      "slot": "breakfast",
      "customItem": {
        "name": "Oatmeal with Banana",
        "amount": 250,
        "unit": "g",
        "nutritionBasis": {
          "amount": 100,
          "unit": "g"
        },
        "macrosPerBasis": {
          "kcal": 150,
          "protein": 5.5,
          "carbohydrate": 27,
          "fat": 2.5
        }
      },
      "createdAt": "2025-11-05T08:30:00.000Z",
      "updatedAt": "2025-11-05T08:30:00.000Z"
    }
    ```
  </Step>

  <Step title="Alternative: Use a recipe">
    Instead of a custom item, you can reference a saved recipe with servings:

    ```json
    {
      "date": "2025-11-05",
      "slot": "lunch",
      "recipeId": "507f1f77bcf86cd799439020",
      "servings": 1.5
    }
    ```

    <Warning>
      Meals must use either `customItem` OR `recipeId + servings` - they're mutually exclusive.
    </Warning>
  </Step>
</Steps>

## View Your Meal Data

Fitboard provides multiple ways to retrieve your meal history:

<CodeGroup>
```bash Get All Meals (Paginated)
curl -X GET "https://api.fitboard.com/api/v1/meals?page=1&limit=20" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Get Meals for a Specific Day
curl -X GET "https://api.fitboard.com/api/v1/meals/day?date=2025-11-05" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Get Meals in a Date Range
curl -X GET "https://api.fitboard.com/api/v1/meals/range?from=2025-11-01&to=2025-11-07" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```
</CodeGroup>

The response includes all your meals with their complete nutrition data:

```json
{
  "date": "2025-11-05",
  "items": [
    {
      "_id": "507f1f77bcf86cd799439012",
      "userId": "507f1f77bcf86cd799439011",
      "date": "2025-11-05",
      "slot": "breakfast",
      "customItem": {
        "name": "Oatmeal with Banana",
        "amount": 250,
        "unit": "g",
        "macrosPerBasis": {
          "kcal": 150,
          "protein": 5.5,
          "carbohydrate": 27,
          "fat": 2.5
        }
      }
    }
  ]
}
```

## Track Your Progress

Beyond meals, Fitboard helps you track body metrics and progress photos.

<Steps>
  <Step title="Create a progress entry">
    Log your weight, measurements, and other metrics:

    ```bash
    curl -X POST https://api.fitboard.com/api/v1/progress \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_JWT_TOKEN" \
      -d '{
        "date": "2025-11-05",
        "weight_kg": 72.5,
        "waist_cm": 82,
        "body": {
          "bodyFatPct": 18.5
        },
        "activity": {
          "steps": 8500
        },
        "notes": "Feeling great this week!"
      }'
    ```
  </Step>

  <Step title="View your progress stats">
    Get aggregated statistics over time:

    ```bash
    curl -X GET "https://api.fitboard.com/api/v1/progress/stats?from=2025-10-01&to=2025-11-05" \
      -H "Authorization: Bearer YOUR_JWT_TOKEN"
    ```

    This returns your progress deltas and weekly averages:

    ```json
    {
      "count": 35,
      "range": {
        "from": "2025-10-01",
        "to": "2025-11-05"
      },
      "weight": {
        "start": 75.0,
        "end": 72.5,
        "delta": -2.5,
        "perWeek": -0.71
      },
      "waist": {
        "start": 86,
        "end": 82,
        "delta": -4,
        "perWeek": -1.14
      }
    }
    ```
  </Step>
</Steps>
