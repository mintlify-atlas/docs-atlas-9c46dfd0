title: Password Reset
---

The password reset flow consists of two steps: requesting a reset token and using that token to set a new password.

## Step 1: Request Password Reset

<api>POST /api/v1/forgot-password</api>

Initiate the password reset process by requesting a reset token. An email will be sent to the user with a reset link.

### Request

<ParamField body="email" type="string" required>
  Email address of the account to reset. Must be a valid email format.
</ParamField>

### Response

<ResponseField name="ok" type="boolean">
  Always returns `true`, even if the email doesn't exist (to prevent user enumeration)
</ResponseField>

<ResponseField name="preview" type="string">
  In development mode only: URL to preview the reset email
</ResponseField>

<ResponseField name="resetLink" type="string">
  In development mode only: The actual reset link that would be sent via email
</ResponseField>

### Request Example

<RequestExample>

```bash cURL
curl -X POST https://api.fitboard.com/api/v1/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane@mail.com"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.fitboard.com/api/v1/forgot-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'jane@mail.com'
  })
});

const data = await response.json();
```

</RequestExample>

### Response Example

<ResponseExample>

```json 200 Success (Production)
{
  "ok": true
}
```

```json 200 Success (Development)
{
  "ok": true,
  "preview": "https://ethereal.email/message/XYZ123",
  "resetLink": "http://localhost:5173/auth/reset-password?token=abc123def456&email=jane%40mail.com"
}
```

```json 400 Validation Error
{
  "error": "Validation failed",
  "issues": [
    {
      "code": "invalid_string",
      "validation": "email",
      "path": ["email"],
      "message": "Invalid email"
    }
  ]
}
```

</ResponseExample>

### How It Works

1. User submits their email address
2. If the email exists in the database:
   - A random reset token is generated
   - The token is hashed (SHA-256) and stored in the database
   - Token expiry is set to 30 minutes from now
   - An email is sent with a reset link containing the token
3. Response always returns `{"ok": true}` to prevent email enumeration

## Step 2: Reset Password with Token

<api>POST /api/v1/reset-password</api>

Complete the password reset by providing the token received via email and a new password.

### Request

<ParamField body="token" type="string" required>
  The reset token received via email. Must be at least 1 character.
</ParamField>

<ParamField body="email" type="string" required>
  Email address of the account. Must match the email associated with the token.
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new password to set. Must be at least 8 characters long.
</ParamField>

### Response

<ResponseField name="ok" type="boolean">
  Indicates successful password reset
</ResponseField>

### Request Example

<RequestExample>

```bash cURL
curl -X POST https://api.fitboard.com/api/v1/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "abc123def456ghi789",
    "email": "jane@mail.com",
    "newPassword": "NewSecurePass123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.fitboard.com/api/v1/reset-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    token: 'abc123def456ghi789',
    email: 'jane@mail.com',
    newPassword: 'NewSecurePass123'
  })
});

const data = await response.json();
```

</RequestExample>

### Response Example

<ResponseExample>

```json 200 Success
{
  "ok": true
}
```

```json 400 Invalid or Expired Token
{
  "error": "Invalid or expired token"
}
```

```json 400 Validation Error
{
  "error": "Validation failed",
  "issues": [
    {
      "code": "too_small",
      "minimum": 8,
      "path": ["newPassword"],
      "message": "String must contain at least 8 character(s)"
    }
  ]
}
```

</ResponseExample>

### How It Works

1. User submits token, email, and new password
2. Token is hashed (SHA-256) and looked up in the database
3. System verifies:
   - Email matches the token's associated user
   - Token hash matches stored hash
   - Token expiry time is in the future (not expired)
4. If valid:
   - New password is hashed and saved
   - Reset token and expiry are cleared from the database
   - Success response is returned
5. If invalid, returns error

## Complete Reset Flow

1. User clicks "Forgot Password" on the login page
2. User enters their email and submits to `POST /api/v1/forgot-password`
3. User receives an email with a reset link (format: `{FRONTEND_URL}/auth/reset-password?token={TOKEN}&email={EMAIL}`)
4. User clicks the link and is taken to the reset password page
5. User enters their new password
6. Frontend submits token, email, and new password to `POST /api/v1/reset-password`
7. Password is updated and user can log in with new credentials

## Security Features

- Reset tokens are hashed (SHA-256) before storage
- Tokens expire after 30 minutes
- Token must match both the hash and the associated email
- Successful reset clears the token to prevent reuse
- Email enumeration is prevented by always returning success from the forgot-password endpoint
- Tokens are single-use (cleared after successful reset)
