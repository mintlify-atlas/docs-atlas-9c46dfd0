---
title: Logout
api: POST /api/v1/logout
---

Log out the current user by clearing the authentication session and cookies.

## Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header.

```
Authorization: Bearer {your_jwt_token}
```

## Request

No request body required.

## Response

<ResponseField name="ok" type="boolean">
  Indicates successful logout
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST https://api.fitboard.com/api/v1/logout \

```

## What Happens on Logout

1. The `auth_token` cookie is cleared with the following options:
   - Path: `/`
   - SameSite: `none`
   - Secure: `true`

2. The session object is nullified

3. A success response is returned

## Client-Side Actions

After successfully logging out, the client should:

- Remove the stored JWT token from local storage or session storage
- Clear any cached user data
- Redirect to the login page or home page
- Remove the Authorization header from future requests

## Notes

- The logout endpoint clears server-side session state
- Even after logout, the JWT token itself remains valid until its expiration time
- Clients should discard the token immediately after logout
- The endpoint returns success even if no active session exists
